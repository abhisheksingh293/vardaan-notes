# Automated PDF Splitting & Organization Report

This report documents the systematic approach used to split 58 chapters from three primary textbooks (Maths, Science, and Social Science) for Class 08 and organize them into their respective chapter-wise subdirectories.

---

## 🚀 High-Level Workflow
The task was executed in three distinct phases for each subject:

1. **Digital Analysis (Chrome Browser)**: Using a browser subagent to identify the Table of Contents (TOC) and correlate printed page numbers with absolute PDF page indices.
2. **Scripted Extraction (Node.js)**: Developing a custom splitting script utilizing the `pdf-lib` library.
3. **Automated Organization**: Saving the extracted chapters directly into the pre-defined chapter-level folders.

---

## 🔍 Phase 1: Browser-Based Identification
Since PDF files often contain cover pages, prefaces, and introductory material before the actual content starts, there is almost always a **Page Offset**.

### How Chrome Was Used:
I utilized an automated browser tool to:
- Open each textbook PDF.
- Locate the **Table of Contents**.
- Perform a "Sync Check": I cross-referenced a printed page number from the TOC (e.g., Chapter 1 starts on page 1) with the browser's current PDF page index (e.g., the browser says I am on page 3).
- **Resulting Offsets**:
    - **Maths**: +1 (Printed Page 1 = Absolute PDF Page 2)
    - **Science**: +2 (Printed Page 1 = Absolute PDF Page 3)
    - **SST**: +1 (Printed Page 1 = Absolute PDF Page 2)

## 🛠️ Phase 2: The Splitting Engine
I used **Node.js** as the runtime environment for the following reasons:
- **Speed**: Node.js handles large binary streams (like 150MB PDFs) very efficiently.
- **Library Support**: `pdf-lib` is a robust library that allows for page extraction without re-encoding, preserving the original quality and layer structure of the PDF.

### Integration Steps for Your Developer:
1. **Initialize Environment**:
   ```bash
   npm init -y
   npm install pdf-lib
   ```
2. **Execution Script**: Below is the generalized script used for the extraction.

```javascript
const { PDFDocument } = require('pdf-lib');
const fs = require('fs');
const path = require('path');

async function splitPdf() {
    const srcPath = 'class_textbook.pdf'; // Path to source PDF
    const data = fs.readFileSync(srcPath);
    const srcDoc = await PDFDocument.load(data);

    // This array contains the identified ranges from the TOC
    const chapters = [
        { name: 'C1_Chapter_Name', range: [3, 18] }, // [Start, End] inclusive
        // ... add all 20+ chapters here
    ];

    for (const chapter of chapters) {
        const newDoc = await PDFDocument.create();
        
        // Convert to 0-indexed for the library
        const start = chapter.range[0] - 1; 
        const end = chapter.range[1]; 
        
        const indices = [];
        for (let i = start; i < end; i++) {
            indices.push(i);
        }

        // Extract and copy the specific pages
        const copiedPages = await newDoc.copyPages(srcDoc, indices);
        copiedPages.forEach(page => newDoc.addPage(page));

        // Save to the specific subdirectory
        const pdfBytes = await newDoc.save();
        const targetFolder = chapter.name;
        
        // Ensure folder existence
        if (!fs.existsSync(targetFolder)) {
            fs.mkdirSync(targetFolder);
        }
        
        const outputFileName = path.join(targetFolder, `${chapter.name}.pdf`);
        fs.writeFileSync(outputFileName, pdfBytes);
        console.log(`Successfully Saved: ${outputFileName}`);
    }
}

splitPdf().catch(console.error);
```

## ⚠️ Important Developer Notes
1. **Memory Management**: For PDFs larger than 200MB, ensure the Node.js heap limit is sufficient or use a streaming extraction method.
2. **Page Range Validation**: Always verify the last page of the last chapter against the total PDF page count to ensure No "orphan" pages are left behind.
3. **Naming Convention**: The script assumes the subdirectory name matches the `name` property in the `chapters` array.
