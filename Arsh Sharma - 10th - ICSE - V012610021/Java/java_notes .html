Chapter 1: The World of Java

What is Java?

Imagine you want to build a house, a robot, or a video game. To do that, you need a set of instructions. Java is a popular, easy-to-learn language that we use to give instructions to a computer.

Fun Fact: Java was created by James Gosling at Sun Microsystems in 1995. At first, it was called "Oak", but they later changed it to Java (inspired by Java coffee!).

Where is Java used?

Java is everywhere! Here are some popular things built with Java:

  • Minecraft: The original PC version of Minecraft was made entirely in Java.
  • Android Apps: Almost all apps on Android phones use Java.
  • Space & Banking: NASA and big banks use Java because it is very safe and reliable.

How does Java work? (JDK, JRE, JVM)

To understand Java, you need to know three important "toolboxes":

Name What it stands for Simple Explanation
JDK Java Development Kit The Builder's Toolkit. It contains everything you need to write and build Java code. If you are coding, you need the JDK.
JRE Java Runtime Environment The Playground. It contains the libraries needed to run a Java program. (If you just want to play Minecraft, you only need the JRE).
JVM Java Virtual Machine The Engine/Translator. Your computer only understands 0s and 1s. The JVM translates your Java code into a language your specific computer understands.

Why is Java called "Robust" and "WORA"?

  • Robust (Strong): Java rarely crashes. It has "Garbage Collection," meaning it automatically cleans up computer memory when you don't need it anymore.
  • WORA (Write Once, Run Anywhere): If you write Java code on a Windows laptop, you can run that exact same code on an Apple Mac or a Linux computer. You don't have to rewrite it!

How to start coding? (Windows & Mac)

To write Java, you need a special software called an IDE (Integrated Development Environment). It is like MS Word, but for coding!

  • Popular IDEs: IntelliJ IDEA, Eclipse, or Visual Studio Code (VS Code).
  • You simply download the JDK from the Oracle website, install an IDE, and you are ready to code on both Windows and Mac!

Chapter 2: Anatomy of a Java Program

Let's look at the most famous piece of code: The "Hello World" program. We will break it down word-by-word so you understand exactly what is happening.

public class HelloWorld {
    public static void main(String[] args) {
        // This is a comment. It tells the computer to print a message.
        System.out.println("Hello, Vardaan Students!");
    }
}

Breaking down the magic words:

Word What does it mean?
public Think of this as an "Open Door." It means anyone (or any other program) can see and use this part of the code.
class A class is a blueprint. Just like a blueprint is used to build a house, a class is used to build our program. Our class name here is HelloWorld.
static Normally, to use something in a class, we have to create an "Object" first. `static` means we don't need an object to run this. It's ready to go immediately!
void `void` means "empty" or "nothing." It tells the computer: "Hey, when this code finishes running, it won't give back any answer or number."
main This is the Entry Gate. When you click "Run", the computer immediately looks for the word `main` and starts executing from there. No `main`, no run!
String[] args This is a way to pass secret messages to your program right when it starts from the command line.
// Comments Anything after // is ignored by the computer. It's just a sticky note left by the programmer for humans to read.
System.out.println System: Means your computer system.
out: Means the output screen (monitor).
println: Means "print this text, then move to a new line."

Chapter 3: Variables, Data Types & ASCII

What is a Variable?

Imagine a variable as a labeled box where you store things.

If you want to store Advik's age (12), you need a box labeled "Age" that can hold numbers. If you want to store his name, you need a box labeled "Name" that can hold text.

Data Types (The size and shape of the box)

1. Primitive Data Types (Simple, basic boxes)

  • int: Stores whole numbers. Example: int age = 15;
  • double: Stores decimal numbers. Example: double weight = 45.5;
  • char: Stores a single character. Example: char grade = 'A'; (Always use single quotes).
  • boolean: Stores true or false. Example: boolean isStudent = true;

2. Non-Primitive Data Types (Complex boxes)

These boxes can hold complex objects. The most common one is a String.

  • String: Stores a sentence or word. Example: String name = "Mayank"; (Always use double quotes).

Type Casting (Changing the box)

Sometimes you need to convert one type to another. Think of pouring water from a cup to a bucket.

  • Implicit (Automatic): Pouring a small cup (`int`) into a big bucket (`double`). Java does this automatically safely.
  • Explicit (Manual): Pouring a big bucket (`double`) into a small cup (`int`). Some water might spill (decimals are lost), so you have to force Java to do it.
double exactMarks = 95.5;
// Forcing a double to become an int (The .5 is lost forever!)
int roundedMarks = (int) exactMarks; 
System.out.println(roundedMarks); // Prints 95

What are ASCII Values?

Computers only understand numbers (0s and 1s). They don't understand letters. So, Java uses ASCII (a secret codebook) to turn characters into numbers!

Important ASCII to remember:
Capital 'A' = 65 (B=66, C=67...)
Small 'a' = 97 (b=98, c=99...)

Chapter 4: Operators & User Input

Programs are boring if they don't do math or talk to the user! Let's learn how to do both.

1. Operators (The Math and Logic Symbols)

Operators are special symbols that perform specific operations on one, two, or three variables, and then return a result. Let's look at the 5 main types!

A. Arithmetic Operators (Math Time)

Used for basic math: + (Add), - (Subtract), * (Multiply), / (Divide).

The Modulus Operator `%`: This is the "Remainder" operator.
10 % 3 = 1 (Because 10 divided by 3 leaves a remainder of 1). It is super useful to check if a number is Even or Odd!

B. Assignment Operators (Giving Values)

Used to assign or update values in a variable. The most common is =.

int score = 50; // Assigns 50 to score
score += 10;    // Shortcut for: score = score + 10 (Now score is 60)
score -= 5;     // Shortcut for: score = score - 5 (Now score is 55)

C. Relational (Comparison) Operators

These compare two values and always answer with a Boolean (true or false). Very useful in decision making!

  • == : Equal to (Notice it is DOUBLE equals. Single = is for assignment!)
  • != : Not equal to
  • > and < : Greater than / Less than
  • >= and <= : Greater than or equal to / Less than or equal to
int rishavAge = 15;
System.out.println(rishavAge == 15); // Prints true
System.out.println(rishavAge > 18);  // Prints false

D. Logical Operators (Combining Conditions)

Used to check multiple conditions at the exact same time.

  • && (Logical AND): True ONLY if BOTH sides are true. (e.g., hasTicket && isVIP)
  • || (Logical OR): True if AT LEAST ONE side is true. (e.g., isWeekend || isHoliday)
  • ! (Logical NOT): Reverses the answer. (e.g., !true becomes false).

E. Unary Operators (Increment/Decrement)

These are super quick shortcuts to add or subtract exactly 1 from a variable. We use them a lot in loops!

  • ++ (Increment): Adds 1. x++ is the exact same as x = x + 1
  • -- (Decrement): Subtracts 1. x-- is the exact same as x = x - 1

2. Getting User Input (`Scanner`)

How do we ask the user a question and read their answer? We use a built-in Java tool called the Scanner class!

import java.util.Scanner; // 1. We must import the Scanner tool

public class InputGame {
    public static void main(String[] args) {
        // 2. Create the Scanner object (Tells Java to listen to the keyboard)
        Scanner sc = new Scanner(System.in);
        
        System.out.println("Hello! What is your name?");
        // 3. Wait for the user to type a String and press Enter
        String studentName = sc.nextLine(); 
        
        System.out.println("What class are you in?");
        // 4. Wait for the user to type a Number
        int studentClass = sc.nextInt(); 
        
        System.out.println("Awesome! Welcome to class " + studentClass + ", " + studentName + "!");
    }
}

Chapter 5: Control Flow (Decisions & Loops)

Control flow allows our program to be smart. It lets the program decide what to do based on the situation.

1. Making Decisions

The if-else statement

int marks = 85;
if (marks >= 90) {
    System.out.println("Aradhya got an A+!");
} else if (marks >= 80) {
    System.out.println("Aradhya got an A.");
} else {
    System.out.println("Aradhya passed.");
}

The switch statement

Use this when you have many exact choices (like a restaurant menu).

int day = 3;
switch(day) {
    case 1: System.out.println("Monday"); break;
    case 2: System.out.println("Tuesday"); break;
    case 3: System.out.println("Wednesday"); break; // This will print!
    default: System.out.println("Invalid day");
}

2. Looping (Doing things repeatedly)

The `for` loop

Use when you know exactly how many times you want to repeat.

// Start at 1; keep going if <= 5; increase by 1 each time
for (int i = 1; i <= 5; i++) {
    System.out.println("Running... " + i);
}

The `while` and `do-while` loops

while: Repeats until a condition becomes false.
do-while: Runs the code at least ONCE, then checks the condition.

Superpowers: `break` and `continue`

  • break; - Immediately SMASHES and escapes the loop entirely.
  • continue; - Skips the current round and jumps to the next round of the loop.

Chapter 6: Arrays and Strings

What is an Array?

Remember how a variable is a single box? An array is like an egg carton or a medicine box with compartments. It allows you to store many items of the same type under one single name.

// Creating an array to hold 5 integer scores
int[] scores = new int[5];

// Putting values in the boxes (Note: Counting starts at 0!)
scores[0] = 95; // Box 0
scores[1] = 88; // Box 1

// A faster way to create an array:
String[] students = {"Parul", "Arnav", "Naitik", "Yoganshika"};
System.out.println(students[0]); // This prints "Parul"

2D Arrays (Matrices)

If a normal array is a single line of boxes, a 2D Array is a grid, like an Excel Spreadsheet or a Chessboard! It has rows and columns.

// Creates a grid with 3 rows and 3 columns
int[][] grid = new int[3][3];
grid[0][0] = 1; // Top-left corner
grid[2][2] = 9; // Bottom-right corner

Working with Strings

A String is essentially a sequence of characters combined together.

Golden Rule: In Java, Strings are Immutable. This means once you create a String, you cannot change it. If you try to change it, Java actually deletes the old one and creates a brand new one behind the scenes! 
String school = "Vardaan Learning";

// Cool things you can do with Strings:
System.out.println(school.length());       // Prints 16
System.out.println(school.toUpperCase());  // Prints "VARDAAN LEARNING"
System.out.println(school.charAt(0));      // Prints 'V'

Chapter 7: Object-Oriented Programming (The Basics)

OOP is the most important concept in Java! It changes how we think about code. Instead of writing long lists of instructions, we organize our code into "Objects".

Classes vs. Objects

  • Class (The Blueprint): A blueprint for a Car. It says "A car has 4 wheels, a color, and can drive." It's just an idea.
  • Object (The Real Thing): Using that blueprint, we build a real Red Ferrari. We can also build a Blue BMW. Both are "Objects" from the same class!

Constructors (The Factory Workers)

When we create a new Object using the new keyword, a special method called a Constructor is run. Its job is to "set up" the object (like giving the car its color right when it's built).

The `this` Keyword

this simply means "Myself". It is used inside a class to point to its own variables.

class Student {
    String name; // Instance variable
    
    // This is the Constructor! It has the exact same name as the Class.
    public Student(String name) {
        // "this.name" means the variable of THIS specific object
        this.name = name; 
    }
}

public class Main {
    public static void main(String[] args) {
        // The constructor is called here!
        Student s1 = new Student("Swati Priya");
    }
}

Access Modifiers (Security Guards)

We use these to protect our variables from being messed with by other code:

  • public: Like a public park. ANY file in your project can access it.
  • private: Like a secret diary. Only the class itself can read or change it. (This is called Encapsulation).
  • protected: Like a family heirloom. Only the class and its "children" classes can use it.
  • default (No keyword): Like a neighborhood group. Only classes in the exact same folder (package) can see it.

Chapter 8: Deep Dive into OOP (The 4 Pillars)

1. Encapsulation (Hiding Data)

We wrap our data (variables) inside a class and make them private. If someone wants to change the variable, they must use a public "Setter" method. This stops people from setting an age to -50!

2. Inheritance (Parent and Child)

Just like a child inherits traits from their parents, a new Class can inherit variables and methods from an existing Class using the keyword extends. This saves us from rewriting code!

The `super` Keyword: Just like `this` means "Myself", the word `super` means "My Parent". A child class uses `super` to talk to its parent class! 
class Animal {
    void eat() { System.out.println("Eating..."); }
}
// Dog "inherits" from Animal
class Dog extends Animal {
    void bark() { System.out.println("Woof!"); }
}

3. Polymorphism (Many Forms)

A person can have many forms: A woman can be a Mother, a Teacher, and a Sister. In Java, methods can do different things based on the situation.

Method Overloading (Same class) Method Overriding (Parent-Child)
Multiple methods with the same name, but different inputs. A child class writes its own version of a parent's method. 
void add(int a, int b) 
void add(int a, int b, int c)
 
Parent: void sound() { print("?"); }
Child: void sound() { print("Meow"); }

4. Abstraction (Hiding Complexity)

When you drive a car, you press the accelerator, and it moves. You don't need to know how the engine works inside. Abstraction means showing only the essential features. We do this using:

  • Abstract Classes: A half-finished blueprint. It cannot be used to make an object directly; a child MUST finish it.
  • Interfaces: A strict contract. It just provides a list of empty methods that a class `implements` (promises to fulfill).

Chapter 9: Handling Errors (Exceptions)

Sometimes, things go wrong. Maybe the internet disconnects, or the user types a word when we asked for a number. If we don't handle these "Exceptions", our program will crash!

The Try-Catch-Finally Block

We put risky code inside a try block. If an error happens, Java "throws" an exception, and we "catch" it.

public class ErrorExample {
    public static void main(String[] args) {
        try {
            // Risky code: You cannot divide a number by zero!
            int answer = 10 / 0; 
        } catch (Exception e) {
            // This runs if there is an error
            System.out.println("Oops! Math error. You can't divide by zero.");
        } finally {
            // This ALWAYS runs, no matter what happens! (Good for cleanup)
            System.out.println("Finished doing math.");
        }
    }
}

Chapter 10: The Collections Framework

Arrays are great, but their size is fixed. If you create an array of size 5, you cannot add a 6th item. The Collections Framework provides special classes that can grow and shrink automatically!

Important: Wrapper Classes!
Collections only work with Objects, not primitive data types. So, instead of `int`, we use its object wrapper `Integer`. Instead of `double`, we use `Double`.

1. ArrayList (A flexible Array)

import java.util.ArrayList;

public class ListNotes {
    public static void main(String[] args) {
        // Notice we use , not 
        ArrayList<Integer> scores = new ArrayList<>();
        scores.add(95);
        scores.add(100);
        System.out.println(scores.get(0)); // Prints 95
    }
}

2. HashMap (A Dictionary)

A HashMap stores data in Key-Value pairs. You use the Key to find the Value.

import java.util.HashMap;

public class MapNotes {
    public static void main(String[] args) {
        // Key = Roll Number, Value = Name
        HashMap<Integer, String> register = new HashMap<>();
        
        register.put(101, "Sripad");
        register.put(102, "Shreya");
        
        System.out.println(register.get(102)); // Prints Shreya
    }
}

Chapter 11: File Handling (Saving Data)

If you close a Java program, all your variables are erased. If you want to save data permanently (like a high score in a game), you need to write it to a File on your computer.

import java.io.FileWriter; // Tool to write to a file
import java.io.IOException; // Tool to handle file errors

public class FileWriteExample {
    public static void main(String[] args) {
        try {
            // Creates a file named secret.txt on your computer
            FileWriter writer = new FileWriter("secret.txt");
            
            // Write a message for Porshiya
            writer.write("Porshiya says Java is fun!");
            
            // Always close the file when done!
            writer.close(); 
            System.out.println("Successfully wrote to the file.");
        } catch (IOException e) {
            System.out.println("An error occurred making the file.");
        }
    }
}

Chapter 12: Multithreading (Doing tasks together)

Normally, Java executes code line by line. It does Task A, finishes, then does Task B.

Multithreading allows Java to do Task A and Task B at the exact same time. Think of playing a video game: One thread is playing the background music, another thread is moving your character, and another is drawing the enemies.

// Create a small task (A Thread)
class MusicTask extends Thread {
    public void run() {
        System.out.println("Playing background music...");
    }
}

public class Game {
    public static void main(String[] args) {
        MusicTask audio = new MusicTask();
        audio.start(); // This starts the task in the background!
        
        System.out.println("Playing the game...");
    }
}

Chapter 13: Modern Java (Lambdas & Streams)

When you get older and start writing complex code, you will use features introduced in modern versions of Java to make your code shorter and cleaner.

1. Lambda Expressions `->`

A Lambda is a shortcut for writing a method. It uses the arrow symbol `->` to say "Take this input, and do this with it."

2. Streams API

Imagine you have a list of 100 student scores, and you want to find only the scores above 90, and then sort them. Using loops takes many lines of code. Streams let you do it in one single, readable sentence like a pipeline!

import java.util.Arrays;
import java.util.List;

public class ModernJava {
    public static void main(String[] args) {
        List<Integer> scores = Arrays.asList(45, 92, 78, 98, 85);
        
        // The Stream Pipeline: Filter high scores, and print them
        scores.stream()
              .filter(score -> score >= 90)  // Only keep 90 and above
              .forEach(s -> System.out.println("Excellent Score: " + s));
              
        // Output: 
        // Excellent Score: 92
        // Excellent Score: 98
    }
}
